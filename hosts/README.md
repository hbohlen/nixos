# Host Configurations

This directory contains host-specific configurations for different machines in the NixOS setup.

## Directory Structure

```
hosts/
├── desktop/           # Desktop workstation configuration
├── laptop/            # Laptop configuration  
├── server/            # Server configuration
└── README.md          # This file
```

Each host directory contains:
- `default.nix` - Main host configuration
- `hardware-configuration.nix` - Hardware-specific settings (generated by nixos-generate-config)
- `hardware/` - Hardware-specific modules
  - `disko-layout.nix` - Disk partitioning and ZFS layout built from shared templates
  - `disko-zfs.nix` - Disko module integration

## Setting Up a New Host

1. **Hardware Configuration**: Run `nixos-generate-config` on the target machine to generate `hardware-configuration.nix`

2. **Disk Configuration**: Update the arguments in `hardware/disko-layout.nix`:
   ```bash
   # Find your disk device
   lsblk -f
   ls -la /dev/disk/by-id/
   
   # Edit the parameters passed to the shared layout
   mkLayout { device = "/dev/disk/by-id/your-actual-disk-id"; }
   ```

3. **SSH Keys**: Add your SSH public keys to the host configuration:
   ```nix
   users.sshKeys = [ "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5... your-key" ];
   users.enablePasswordAuth = false;  # Disable password auth for security
   ```

4. **Build**: Test the configuration:
   ```bash
   nixos-rebuild build --flake .#hostname
   ```

## Host Types

- **Desktop**: Full desktop environment with development tools
- **Laptop**: Desktop environment optimized for mobile use with power management
- **Server**: Minimal headless configuration without desktop environment

## Security Notes

- Always use SSH keys instead of passwords for authentication
- The initial password "changeme" should be replaced immediately
- Server configurations should disable password authentication
- SSH host keys are persisted in `/persist/etc/ssh/` via impermanence
